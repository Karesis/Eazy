# Eazy 语言规范 - v0.1.0

## 1. 简介

Eazy 是一种命令式、底层风格的编程语言，设计核心为直观 (Intuitive) 与保持简单 (KISS)，其部分灵感来源于 Brainfuck 和汇编语言。它采用全局作用域，提供显式的栈操作、类型化的变量、结构体、独特的控制流机制，并旨在通过极简的核心特性实现表达能力。

## 2. 词法元素

### 2.1. 注释

以 `#` 字符开始，直到行尾的内容被视为单行注释，编译器会忽略它们。

```eazy
# 这是一个单行注释
int64 counter # 这是行尾注释
```

### 2.2. 标识符 (Identifiers)

*   用于命名变量、栈、结构体、标签。
*   必须以字母 (a-z, A-Z) 或下划线 (`_`) 开头。
*   后续字符可以是字母、数字 (0-9) 或下划线。
*   区分大小写 (例如，`myVar` 和 `myvar` 是不同的标识符)。
*   关键字（见下文）不能用作标识符。

### 2.3. 关键字

以下标识符被保留为关键字，具有特殊含义：
`stack`, `int8`, `int16`, `int32`, `int64`, `uint8`, `uint16`, `uint32`, `uint64`, `push`, `pop`, `goto`, `back`, `if`, `and`, `or`, `not`, `add`, `sub`, `mul`, `div`, `mod`, `print`, `halt`, `main` (作为默认入口标签)

### 2.4. 整数立即数 (Literals)

*   只支持十进制整数。
*   可以带可选的负号 (`-`)。
*   默认为 `int64` 类型。

### 2.5. 符号与分隔符

*   `:` : 用于标签定义。
*   `=` : 用于赋值。
*   `.` : 用于访问结构体字段。
*   `{`, `}` : 用于定义结构体。
*   `,` : (可选) 用于分隔结构体字段定义。
*   `(` `)` : 用于运行时计算表达式和控制 `if` 条件优先级。
*   `[` `]` : 用于栈解引用。
*   空白符 (空格, 制表符, 换行符): 用于分隔词法单元，多个连续空白符通常视为一个。

## 3. 数据类型

### 3.1. 原始整数类型

*   `int8`, `int16`, `int32`, `int64`: 有符号整数。
*   `uint8`, `uint16`, `uint32`, `uint64`: 无符号整数。

### 3.2. 栈 (Stack)

*   通过 `stack <栈名称>` 声明。
*   创建一个全局的、具名的栈区域。
*   **同时**创建一个与栈同名的 `int64` 类型的全局变量，用于存储该栈**当前栈顶元素的索引**。
*   栈索引从 0 开始向上增长 (0, 1, 2, ...)。
*   栈为空时，关联的栈顶索引变量值为 -1。
*   所有栈默认存储 `int64` 类型的值。

### 3.3. 结构体 (Struct)

*   通过 `{}` 定义一个新的复合数据类型，并**立即声明一个该类型的全局实例**。
*   **推荐语法:**
    ```eazy
    <结构体实例名> {
        <类型1> <字段名1>
        <类型2> <字段名2>
        # ... 可以换行 ...
    }
    # 紧凑形式: <实例名> { <类型1> <字段1>, <类型2> <字段2> }
    ```
*   **允许但非推荐语法:**
    ```eazy
    { <类型1> <字段1>, <类型2> <字段2> } <结构体实例名>
    ```
*   结构体可以嵌套包含其他已定义的结构体实例。
*   字段通过 `<实例名>.<字段名>` (例如 `player.position.x`) 访问。

## 4. 变量与作用域

*   所有变量（包括原始类型、栈关联变量、结构体实例）都是**全局作用域**。
*   变量在使用前必须先通过 `intX/uintX`, `stack`, 或 `{}` 声明。
*   变量默认初始化为 0（或对应类型的零值）。栈关联变量初始值为 -1。

## 5. 值与表达式

指令可以操作的值 (Value) 来自以下来源：

*   **立即数 (Immediate):** 如 `10`, `-5`。
*   **变量名 (Variable):** 如 `counter`, `flag`。
*   **栈顶索引变量 (Stack Pointer Variable):** 如 `S` (其值为栈 S 的当前栈顶索引)。
*   **结构体字段 (Struct Field):** 如 `point.x`, `player.position.y`。
*   **运行时计算值 (Computed Value):** 使用 `()` 包裹的表达式，在运行时计算结果。表达式内可包含其他值来源及 `+`, `-` 运算符。例如 `(a + 1)`, `(player.health - 10)`。
*   **解引用值 (Dereferenced Value):** 使用 `[]` 访问指定栈在特定索引处存储的值。
    *   语法: `[<栈名称> : <索引表达式>]`
    *   `<索引表达式>`: 可以是立即数、变量、栈顶索引变量、结构体字段、计算值 `()`。
    *   **求值规则:** 设 `Base` 为 `:` 左侧计算出的基址值，`Offset` 为 `:` 右侧计算出的偏移量值。该表达式访问 `<栈名称>` 中**索引为 `Base - Offset`** 的元素。
    *   **运行时错误:** 如果计算出的最终索引为负数，或大于等于当前该栈的深度 (即大于栈顶索引)，则触发运行时错误并终止程序。

## 6. 语句与指令

### 6.1. 声明语句

*   `stack <栈名称>`
*   `<类型> <变量名>` (如 `int64 counter`)
*   `<结构体实例名> { ... }`

### 6.2. 栈操作

*   **`push <值> <栈名称>`**
    *   计算 `<值>`。
    *   如果 `<值>` 是结构体实例名，则按字段**声明顺序**依次将各字段的值压入 `<栈名称>`。
    *   如果 `<值>` 是普通值，则将其 (扩展为 `int64`) 压入 `<栈名称>`。
    *   更新关联的栈顶索引变量。
*   **`pop <目标> <栈名称>`**
    *   如果 `<目标>` 是结构体实例名，则按字段**声明逆序**依次从 `<栈名称>` 弹出值，存入对应字段（执行截断）。
    *   如果 `<目标>` 是普通变量名，则从 `<栈名称>` 弹出一个 `int64` 值，截断后存入该变量。
    *   更新关联的栈顶索引变量。
    *   从空栈 `pop` 是未定义行为（建议运行时错误）。

### 6.3. 赋值语句

*   **`<目标> = <值>`**
    *   `<目标>`: 变量名或结构体字段。
    *   计算 `<值>`。
    *   将计算结果（必要时截断）存入 `<目标>`。

### 6.4. 控制流

*   **`<标签名>:`**
    *   定义一个代码位置标签。
*   **`goto <标签名>`**
    *   将**下一条指令**的位置压入一个内部的、理论上无限的“返回点栈”。
    *   无条件跳转到 `<标签名>`。
*   **`back`**
    *   从“返回点栈”弹出一个位置。
    *   无条件跳转到该位置。
    *   **运行时错误:** 如果返回点栈为空时执行 `back`，则触发运行时错误并终止程序。
*   **`if <条件表达式>`**
    *   计算 `<条件表达式>` 的值。表达式内可以使用值、逻辑运算符 (`and`, `or`, `not`) 和括号 `()`。**（注：需要明确运算符优先级和结合性）**
    *   如果结果**非零 (true)**，则**执行紧随其后的下一行指令**。
    *   如果结果为**零 (false)**，则**跳过紧随其后的下一行指令**，从再下一行开始执行。

### 6.5. 算术/逻辑运算

*   **格式:** `OP <源值> <目标>`
*   **`<目标>`:** 变量名或结构体字段。
*   **语义:** `目标 = 目标 <Operator> 源值` (对于二元运算)。
*   **操作 (OP):** `add`, `sub`, `mul`, `div`, `mod`, `and`, `or`, `xor`。
*   **一元操作:** `not <目标>` (语义: `目标 = ~目标`)。
*   **类型处理:** 运算通常在 `int64` 级别执行，结果根据 `<目标>` 类型截断。
*   **运行时错误:** `div` 或 `mod` 的 `<源值>` 为零时，触发运行时错误并终止程序。

### 6.6. 输出

*   **`print <值>`**
    *   计算 `<值>`。
    *   将其整数表示打印到标准输出，后跟换行符。

### 6.7. 程序终止

*   **`halt`**
    *   立即终止程序执行。

## 7. 程序结构

*   Eazy 程序由一系列语句组成。
*   默认执行入口点是标签 `main:`。如果不存在 `main:` 标签，程序行为未定义（建议编译器报错）。
*   程序执行直到遇到 `halt` 指令，或发生未处理的运行时错误，或执行流自然结束（例如从 `main` 的末尾掉出）。

## 8. 已定义的运行时错误

以下情况会导致程序运行时错误并终止：

1.  **解引用越界:** `[]` 计算出的索引为负数，或大于等于目标栈的当前深度。
2.  **空栈返回:** 执行 `back` 指令时，内部返回点栈为空。
3.  **除零/模零:** `div` 或 `mod` 指令的 `<源值>` 计算结果为零。
4.  **(建议) 空栈弹栈:** 执行 `pop` 指令时，目标栈为空。

## 9. 注意事项与待明确

*   **`if` 条件表达式中的运算符优先级和结合性**需要明确定义 (例如，`not` > `and` > `or`？从左到右结合？)。
*   标识符的具体字符集规则 (是否支持 Unicode 等) 可以在实现时确定。
*   `pop` 空栈的行为建议明确为运行时错误。
