# Eazy 语言和 Claw 编译器

## 项目简介

**Eazy** 是一门正在实验和开发中的编程语言，其核心理念是“万物皆块（Everything is a Block）”。它旨在提供一种新颖且底层可控的编程范式，探索不同于传统结构化编程和面向对象编程的代码组织与控制流方式。

**Claw** 是 Eazy 语言的第一个编译器，目前正在早期开发阶段。它的短期目标是将 Eazy 语言源代码编译（或转译）为 Python 代码，以便快速迭代和验证语言设计。长远目标是实现自编译，并能够编译出用于开发系统内核的低级代码。

本项目灵感部分来源于 C 语言、Python 以及汇编语言，并希望借鉴 GNU 计划的精神，从底层构建一个完整的生态系统（语言、编译器、内核、shell等），尽管目前仍在非常初期的探索阶段。

## 当前状态

本项目目前处于 **非常早期** 的开发阶段。

已完成的工作包括：

*   **词法分析器 (Lexer):** 能够将 Eazy 语言源代码分解为 Token 序列。
*   **语法分析器 (Parser V1):** 能够识别顶层块定义（`@name:`）以及基础的内部块（`goto`, `back`, 和通用的行块），并构建初步的抽象语法树 (AST)。当前版本的 Parser 对行块（如赋值、打印）不进行深度解析，而是作为通用行处理。变量声明（`int`）在当前版本中被跳过。

## 核心概念（当前理解）

*   **万物皆块:** 代码的基本组成单元是块。
*   **块间控制流:** 主要通过 `goto` 和 `back` 指令实现块之间的跳转和返回。
*   **命名块 (`@name:`):** 具有名字的顶层块定义。
*   **行块 (Line Block):** 块内部的单行操作，如赋值、打印、控制流指令等。

*(注：这些概念会随着语言设计深入而演进)*

## 如何构建和运行 (当前)

本项目需要 Python 3.6 或更高版本来运行当前的编译器组件。

1.  克隆本仓库：
    ```bash
    git clone <你的 GitHub 仓库 HTTPS 或 SSH 地址>
    cd <你的仓库目录名，例如 Eazy>
    ```
2.  确保你位于项目的根目录下。
3.  Lexer 和 Parser 模块位于 `claw-compiler/` 目录中。
4.  运行 Parser 并测试它处理示例代码的能力：
    ```bash
    python claw-compiler/claw_parser.py
    ```
    (这将运行 `claw_parser.py` 文件末尾的示例代码，输出 Token 列表和解析后的 AST 结构)。
